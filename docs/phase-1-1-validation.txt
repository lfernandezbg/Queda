3. MODELO IMPLEMENTADO
================================================================
IDENTIFICADORES:
- Implementados como @JvmInline value classes sobre String.
- Validación estricta: No permite vacíos ni espacios en blanco.
- Generación: UUID aleatorio mediante factory newId().
- Tipos: HouseholdId, ProductId, StockItemId, LocationId, ShoppingEntryId, DomainEventId.

MODELO DE CANTIDAD:
- Dimensiones: COUNT, MASS, VOLUME.
- Unidades: UNIT, GRAM, KILOGRAM, MILLILITER, LITER.
- Niveles Aproximados: EMPTY(0), ALMOST_EMPTY(1), LOW(2), MEDIUM(3), HIGH(4), FULL(5).
- StockQuantity (Sealed Interface):
  * ExactQuantity: Usa BigDecimal, normalización de ceros finales, máximo 3 decimales significativos.
  * ApproximateQuantity: Usa ApproximateLevel con propiedad de orden explícita.

RESULTADOS DE DOMINIO:
- DomainResult<T>: Éxito con valor o Fallo con DomainError tipado.
- Errores contemplados: IncompatibleQuantityDimensions, NegativeQuantity, TooManyDecimalPlaces, InsufficientQuantity, ApproximateLevelDidNotDecrease, InvalidQuantityText.

================================================================
4. OPERACIONES DE DOMINIO
================================================================
- convert(): Conversión entre g <-> kg y ml <-> l. Falla si hay pérdida de precisión (>3 decimales).
- add(): Suma de cantidades exactas con conversión automática de la segunda a la primera unidad.
- subtract(): Resta de cantidades exactas. Falla si el resultado es negativo.
- consume(): Alias semántico de subtract() que devuelve InsufficientQuantity si el consumo supera el stock.
- correct(): Permite establecer una nueva cantidad arbitraria (no negativa).
- consumeApproximate(): Exige que el nivel final sea estrictamente inferior al inicial.
- correctApproximate(): Permite cualquier cambio de nivel.
